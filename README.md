# Autobahn Content Hub
### Daily Article Scraping + AI Post Drafts for *Rock Your Business*

Automated daily pipeline: scrapes Google News for articles relevant to each chapter of *Rock Your Business*, generates LinkedIn posts, blog hooks, and email subject lines using Claude AI, and publishes results to a branded HTML dashboard.

---

## What It Does

Every weekday at 7 AM EST, GitHub Actions runs two scripts:

1. **`scripts/scraper.py`** — Pulls Google News RSS for each of the 10 chapter topics and saves results to `data/articles.json`
2. **`scripts/generator.py`** — Sends those articles to the Anthropic API (Claude) and saves three LinkedIn draft posts + a blog hook + email subject per chapter to `data/drafts_latest.json`

The output is committed back to the repo, and `dashboard.html` (optionally hosted on GitHub Pages) reads `drafts_latest.json` to display everything in a browsable, copyable interface.

## Setup (One Time)

### Step 1 — Create a GitHub repo

1. Go to [github.com/new](https://github.com/new)
2. Name it `autobahn-content-hub` (or anything you prefer)
3. Set it to **Private** (recommended)
4. Click **Create repository**

### Step 2 — Upload this folder

Option A — GitHub Desktop (easiest):
1. Download and install [GitHub Desktop](https://desktop.github.com/)
2. Clone your new repo
3. Copy all files from this folder into the cloned folder
4. Commit and push

Option B — Command line:
```bash
cd autobahn-content-hub
git init
git remote add origin https://github.com/YOUR_ORG/autobahn-content-hub.git
git add .
git commit -m "Initial setup"
git push -u origin main
```

### Step 3 — Add your Anthropic API key as a GitHub Secret

1. Go to your repo on GitHub
2. Click **Settings** → **Secrets and variables** → **Actions**
3. Click **New repository secret**
4. Name: `ANTHROPIC_API_KEY`
5. Value: your key from [console.anthropic.com](https://console.anthropic.com)
6. Click **Add secret**

> The key is used only by the GitHub Action and never committed to the repo.

### Step 4 — Enable GitHub Actions

1. Go to the **Actions** tab in your repo
2. If prompted, click **I understand my workflows, go ahead and enable them**
3. You should see **Daily Content Hub** listed

### Step 5 — Run it manually first

1. Click **Daily Content Hub** in the Actions tab
2. Click **Run workflow** → **Run workflow**
3. Watch the run — it takes about 2–3 minutes
4. After it completes, check the `data/` folder in your repo for the output files

## Viewing the Dashboard

### Option A — GitHub Pages (Recommended)

1. Go to **Settings** → **Pages**
2. Source: **Deploy from a branch**
3. Branch: `main`, folder: `/ (root)`
4. Click **Save**
5. Your dashboard will be live at: `https://YOUR_ORG.github.io/autobahn-content-hub/dashboard.html`

### Option B — Open locally

After running the scripts locally (see below), open `dashboard.html` in a browser using a local server:

```bash
python -m http.server 8080
# Then open http://localhost:8080/dashboard.html
```

> Opening the HTML file directly (`file://`) will not work because browsers block local `fetch()` calls for security reasons. Use a local server instead.

## Running Locally

Useful for testing or generating an on-demand batch.

### Prerequisites
- Python 3.10+
- An Anthropic API key

### Setup

```bash
# Clone the repo
git clone https://github.com/YOUR_ORG/autobahn-content-hub.git
cd autobahn-content-hub

# Create a virtual environment (recommended)
python -m venv venv
source venv/bin/activate      # Mac/Linux
venv\Scripts\activate         # Windows

# Install dependencies
pip install -r requirements.txt

# Set up your API key
cp .env.example .env
# Edit .env and add your ANTHROPIC_API_KEY
```

### Run

```bash
# Step 1: Scrape articles
python scripts/scraper.py

# Step 2: Generate posts
python scripts/generator.py

# Step 3: View dashboard
python -m http.server 8080
# Open http://localhost:8080/dashboard.html
```

## Customizing

### Change the schedule

Edit `.github/workflows/daily.yml`:
```yaml
- cron: "0 12 * * 1-5"   # 7 AM EST, Mon-Fri
```
Use [crontab.guru](https://crontab.guru) to build a different schedule.
Remember: GitHub Actions uses UTC. EST = UTC-5 (UTC-4 during daylight saving).

### Add or edit chapter search queries

Edit `CHAPTER_QUERIES` in `scripts/scraper.py`. Each chapter can have up to 3 search queries; the scraper runs all of them and deduplicates results.

### Change the tone or format of posts

Edit `LINKEDIN_PROMPT` in `scripts/generator.py`. The prompt currently generates:
- Post 1: Thought leadership (no book mention)
- Post 2: Article reaction (links to scraped article)
- Post 3: Book tie-in (mentions chapter and authors)

You can add post types, change word counts, or adjust the tone by editing the prompt text.

### Add more output channels

To add email newsletter drafts or Instagram captions, add new fields to `LINKEDIN_PROMPT` in `generator.py` and update the dashboard's `buildCard()` function in `dashboard.html` to display them.

## Troubleshooting

**GitHub Action fails with "ANTHROPIC_API_KEY not set"**
→ Make sure you added the secret in Settings → Secrets → Actions (not in the .env file).

**Dashboard shows "No data yet"**
→ The scripts haven't run yet. Trigger a manual run from the Actions tab, or run locally first.

**Some chapters show "Generation error"**
→ Usually a transient API timeout. Re-run the workflow. If it persists, check the Action logs for the specific error.

**Google News returns no articles**
→ Some queries return fewer results depending on the news cycle. The scraper handles empty results gracefully — generator will still produce posts using the chapter topic context alone.
